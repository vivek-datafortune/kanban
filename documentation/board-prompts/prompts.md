# AI Board Template Prompts

A collection of prompts to use in the **Generate with AI** input on the Templates page. Each is designed to produce a board with a generous number of lists and cards.

---

## Engineering

**Full-stack web app launch**
> Full-stack SaaS product launch with tasks for backend API design, database schema, authentication, frontend components, CI/CD pipeline setup, security audit, performance testing, staging deployment, and production go-live checklist

**Mobile app development cycle**
> Mobile app development for iOS and Android covering product discovery, wireframing, UI design, React Native setup, core features (onboarding, auth, push notifications, offline mode), QA testing, app store submission, and post-launch monitoring

**Bug tracker for a distributed team**
> Bug tracking board for a distributed engineering team with lists for triage, backlog, in-progress, code review, QA verification, and closed — with cards for critical production bugs, regression issues, performance bottlenecks, and technical debt items

**Microservices migration**
> Monolith to microservices migration plan including service boundary analysis, data decomposition, API gateway setup, inter-service communication, containerisation with Docker, Kubernetes orchestration, observability stack, and phased traffic cutover

**Security hardening sprint**
> Application security hardening sprint covering OWASP Top 10 review, dependency vulnerability scanning, secrets rotation, CSP headers, rate limiting, SQL injection fixes, XSS prevention, penetration testing, and compliance documentation

## Product

**Product roadmap — Q3/Q4**
> Quarterly product roadmap with epics for user onboarding redesign, billing and subscription management, mobile app MVP, third-party integrations, analytics dashboard, accessibility improvements, and a customer feedback loop programme

**Feature discovery and validation**
> Feature discovery process from idea intake to launch including user interview scheduling, problem statement definition, opportunity sizing, competitor analysis, prototyping, usability testing, stakeholder sign-off, and go-to-market brief

**OKR planning cycle**
> Company-wide OKR planning for the next two quarters covering leadership alignment sessions, team-level objective drafting, key result definition, dependency mapping, mid-quarter check-ins, and end-of-quarter retrospectives

## Design

> Design system creation including colour palette, typography scale, spacing system, icon library, button variants, form elements, navigation patterns, data display components, accessibility annotations, Storybook documentation, and versioning strategy
